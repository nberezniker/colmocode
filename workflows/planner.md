description: Create a structured plan, break down a task into steps, or strategic planning.
---

This workflow helps in breaking down complex user requests into granular, actionable implementation steps.

### Steps:

1. **Information Gathering**:
   - Analyze the user requirement.
   - Research existing implementations in the codebase to ensure alignment.
   - Identify dependencies and affected components.

2. **Strategy Formulation**:
   - Define the high-level approach.
   - List required changes per component.
   - Identify potential risks or breaking changes.

3. **Plan Output**:
   Generate a structured plan:
   - **Goal Description**: What we are trying to achieve.
   - **Proposed Changes**: Grouped by component/file.
   - **Verification Plan**: How we will test the changes (Automated + Manual).

4. **Integration**:
   Ensure the plan follows the **Senior Architect** standards (Clean code, DRY, scalable).
