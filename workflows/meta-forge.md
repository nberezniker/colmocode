# /meta-forge

Мета-суперспособность Antigravity. Оркестрирует сложные, нечетко сформулированные или масштабные задачи, автоматически подбирая готовые инструменты или динамически создавая новые сценарии работы.

## Входные параметры
- `request`: Текст сложного запроса пользователя.

## Алгоритм работы (Распаковка)

### Фаза 1 — Декомпозиция Намерения
1. **Классификация интента**:
   - `invoke_subagent` [Intent-Classifier]: Анализирует запрос по методологии 4W (Кто, Что, Почему, Когда). Определяет кандидата-агента среди 5 подключенных к MCP (Klava, OpenClaw, Hermes, Benya, Paperclip).
2. **Контекстный Поиск**:
   - `invoke_subagent` [Brain-Context]: Выполняет быстрые `rag_query` к ColmoBrain V3 (задержка ~7с) для получения исторических решений и данных по аналогичным темам.

### Фаза 2 — Маршрутизация
1. **Библиотечный роутер**:
   - `invoke_subagent` [SP-Router]: Сопоставляет декомпозированные задачи с существующей библиотекой суперспособностей `SP-01` .. `SP-09`.
   - Если совпадение найдено → Перенаправляет выполнение в соответствующую суперспособность с нужными параметрами и назначает подходящего MCP-агента.

### Фаза 3 — Динамическое проектирование воркфлоу (Self-Design)
*Выполняется только в случае, если существующие суперспособности не подходят для решения задачи.*
1. **Конструирование**:
   - `invoke_subagent` [Workflow-Designer]: Разрабатывает новый файл воркфлоу `.md` в папке `.agents/workflows/`, сочетая готовые модули и шаблоны.
2. **Проверка безопасности и типов**:
   - `invoke_subagent` [Workflow-Validator]: Убеждается, что новый воркфлоу соответствует правилам изоляции, не создает бесконечных циклов и использует правильные MCP-инструменты.

### Фаза 4 — Выполнение и Обучение
1. **Исполнение**:
   - Запускает выполнение созданного воркфлоу через субагента `[New-SP-Executor]`.
2. **Сохранение опыта**:
   - Выполняет `rag_ingest` в ColmoBrain, добавляя новый спроектированный сценарий в базу знаний для последующего повторного использования.

## Железные правила
- Динамически создаваемые воркфлоу должны быть атомарными и следовать правилу YAGNI.
- Запрещено использовать недокументированные MCP-сервера или сторонние API без верификации.
