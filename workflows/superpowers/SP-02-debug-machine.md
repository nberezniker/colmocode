# /debug-machine

Система автоматического исправления сбоев. Локализует ошибки, выстраивает гипотезы и устраняет их по TDD.

## Входные параметры
- `error`: Текст ошибки или путь к файлу логов.
- `context`: Дополнительный контекст сбоя (необязательно).

## Алгоритм работы (Распаковка)

1. **Классификация сбоя (Smart Switch)**:
   - `invoke_subagent` [Log-Parser]: Извлекает стек вызовов, файлы и тип ошибки.
   - Определяет ветку лечения:
     - *HEAP*: Ошибки OOM/heap.
     - *UI*: Ошибки с таймаутами селекторов в Puppeteer.
     - *TYPES*: Синтаксические или TypeScript ошибки.
     - *GENERAL*: Иные логические ошибки.

2. **Анализ и Гипотезы**:
   - `invoke_subagent` [Hypothesis-Engine]: Генерирует 3 независимые гипотезы причины бага, ранжируя их по вероятности (P1, P2, P3).
   - Выполняет серию быстрых `rag_query` по симптомам в ускоренном ColmoBrain V3 (задержка ~7с, с использованием реранкера), чтобы найти прецеденты и готовые решения.

3. **Локальное воспроизведение (TDD-RED)**:
   - `invoke_subagent` [TDD-Tester]: Пишет минимальный тест, воспроизводящий сбой. Запускает его и подтверждает статус RED.

4. **Патчинг и Проверка (TDD-GREEN)**:
   - `invoke_subagent` [Bug-Fixer]: Делает точечный патч в коде для устранения ошибки (по гипотезе P1).
   - Запускает тест. Если GREEN — переходит к шагу 5.
   - Если RED — откатывает патч и пробует гипотезу P2 (до 3 итераций).

5. **Пост-контроль**:
   - Запускает `npm run type-check` и полный набор тестов для измененного модуля.
   - `invoke_subagent` [Blast-Radius]: Гарантирует, что исправление не затронуло соседние модули.
   - Выполняет `rag_ingest` с отчетом: `[debug] симптом -> причина -> патч`.

## Правила
- Не переходить к исправлению без воспроизводящего теста.
- Не изменять файлы вне области видимости бага (Blast Radius = 1 файл в 90% случаев).
