# /brain-sync

Синхронизатор знаний ColmoBrain. Поддерживает базу знаний в актуальном состоянии, очищает граф от мусора, автоматически парсит изменения из git и архивирует устаревшие архитектурные решения.

## Входные параметры
- `since`: Время, за которое нужно проанализировать историю изменений (по умолчанию `6hours`).

## Алгоритм работы (Распаковка)

### Фаза 1 — Проверка здоровья базы
1. **Пинг RAG**:
   - `invoke_subagent` [Brain-Ping]: Выполняет быстрый `rag_query` со строкой "ping" и измеряет время отклика (latency). Нормальное время отклика ColmoBrain V3 составляет ~7 секунд (локальный реранкер `bge_rerank` и служба `com.lightrag.reranker` отключены). Если отклик > 15 секунд, сигнализирует о задержке сети/API и проводит диагностику, но не пытается запускать прогрев локального реранкера.
2. **Анализ Git коммитов**:
   - `invoke_subagent` [Git-Delta]: Извлекает `git log --since=6hours` для поиска изменений в кодовой базе и документации.

### Фаза 2 — Фильтрация и Экстракция
1. **Экстракция архитектурных изменений (ADR)**:
   - `invoke_subagent` [ADR-Extractor]: Находит новые или измененные архитектурные решения в diff, классифицирует их по уровню влияния (Tier 1/2/3).
2. **Фильтрация шума**:
   - `invoke_subagent` [Garbage-Filter]: Анализирует граф сущностей ColmoBrain. Выявляет устаревшие данные, дубликаты и нерелевантные логи. Составляет drop-лист.

### Фаза 3 — Синхронизация и Отчет
1. **Массовый Ingest**:
   - `invoke_subagent` [Batch-Ingestor]: Выполняет `rag_ingest` для всех отфильтрованных и одобренных документов пакетом.
2. **Очистка**:
   - Удаляет из графа сущности из drop-листа.
3. **Уведомление**:
   - Отправляет статус синхронизации в Telegram-канал benya: "Brain Sync ✓ Добавлено документов: N, Удалено шума: M, Латентность: Xms".

## Железные правила
- Документы с релевантностью ниже `filter_threshold` (0.7) не должны попадать в ColmoBrain.
- Никогда не стирать актуальные ADR без явного перевода их в статус DEPRECATED.
