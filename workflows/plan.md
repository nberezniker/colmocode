# /plan (Implementation Planning)

Use this workflow to design architectural changes, map out features, and get user approval before writing code.

## Workflow Phases

### Phase 1: Requirements Analysis & Grounding
1. Restate the core goal and requirements.
2. Identify code files that will be modified or created.
3. Search the codebase for existing patterns to mirror.

### Phase 2: Design & Task Breakdown
1. Break the task down into clear, granular steps.
2. Structure the tasks using checklist format.
3. Identify dependencies, potential risks, and mitigations.

### Phase 3: Verification Design
1. Specify how each step will be tested (unit tests, types checks, manual verification).
2. Write down the commands to run.

### Phase 4: User Approval
1. Present the implementation plan to the user.
2. **WAIT** for user approval before proceeding to make any code changes.
