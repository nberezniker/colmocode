# /forge

Автономный TDD-инженер. Реализует фичи и исправления с соблюдением стандартов качества проекта ColmoGravity.

## Входные параметры
- `task`: Описание задачи (что нужно реализовать).
- `files`: Затрагиваемые файлы (необязательно, определяется автоматически).

## Алгоритм работы (Распаковка)

1. **Инициализация и контекст**:
   - `invoke_subagent` [Brain-Warmer]: Выполняет `rag_query` по теме задачи для получения ADR и контекста. Благодаря ColmoBrain V3 (ускорение до ~7с, локальный реранкер включен), запросы выполняются быстро и могут повторяться для уточнения контекста.
   - `invoke_subagent` [Scope-Analyzer]: Анализирует `task.md` и `git status`, локализует изменения. Распределяет подзадачи между подключенными через MCP агентами (включая Klava, OpenClaw).

2. **TDD Цикл (RED)**:
   - `invoke_subagent` [TDD-Writer]: Пишет минимальный тест, воспроизводящий задачу или баг.
   - Запускает `vitest run` для проверки падения теста по правильной причине (RED).

3. **Реализация (GREEN)**:
   - `invoke_subagent` [Implementer]: Пишет минимальный рабочий код для прохождения теста.
   - Запускает `vitest run` и убеждается, что тест пройден (GREEN).

4. **Очистка и Контроль Качества**:
   - `invoke_subagent` [Spec-Reviewer]: Проверяет соответствие реализации исходному таску (YAGNI).
   - `invoke_subagent` [Code-Reviewer]: Ищет и чистит `: any`, `console.*` (заменяя на logger), проверяет ADR-001..004.
   - Запускает `npm run type-check` для проверки типов.

5. **Фиксация**:
   - `invoke_subagent` [Git-Patcher]: Проверяет `git diff` на Blast Radius. Если всё в рамках `task.md`, делает коммит.
   - Выполняет `rag_ingest` с отчетом о проделанной работе.

## Правила
- Код без теста удаляется.
- Типизация: строго без `: any`.
- Логирование: строго через `logger` вместо `console.log`.
